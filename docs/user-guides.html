LogisX — User Guides by Role

Complete reference for Super Admin, Dispatcher, Driver, and Investor
🛡
Super Admin

Full system access. Manages operations, users, fleet, finances, and data quality. The Super Admin is the system owner and has unrestricted access to every page and function.

Overview & Sidebar Navigation

After login, the Super Admin lands on /dashboard. The left sidebar provides access to all 12 pages:

The sidebar is collapsible — click the toggle button or use the hamburger icon. Collapse state is saved in localStorage across sessions.

The sidebar also shows a connection status indicator (green dot = connected, red dot = disconnected from WebSocket) and an unread notification badge on the Notifications item.

Operations Dashboard /dashboard

KPI Cards (Top Row)

  • Active Loads — Count of loads in: Dispatched, Assigned, At Shipper, Loading, In Transit, At Receiver, Unloading
  • Unassigned Jobs — Loads with status Unassigned, New, Open, Pending, or empty status with a valid Load ID
  • Completed — Delivered/Completed loads this month
  • Fleet Size — Total trucks in fleet with utilization (assigned vs total)

Clicking a KPI card switches to the corresponding tab below.

Revenue Cards (Super Admin Exclusive)

  • Total Revenue — All payments summed
  • Paid — Revenue with "Paid" status
  • Pending — Unpaid invoices

These cards are hidden from Dispatcher role.

Job Board Tab

Lists all unassigned loads. Each row shows Load ID, origin, destination, pickup date, rate. Use the Driver dropdown to select a driver, then click "Assign" to dispatch the load.

Active Loads Tab

Shows loads currently in progress. For each load you can:

  • Update Status — Select new status from dropdown, click "Go"
  • Reassign — Change driver from dropdown, click "Go"
  • Cancel — Returns load to Unassigned (back to Job Board)
  • View Details — Click load row to see full info with route map

Completed Loads Tab

Historical load data merged from Job Tracking and Job Summary sheets. Search, paginate, and click to view details. Shows completion date, POD status, and full load information.

Fleet & Drivers Tab

Driver cards showing: name, assigned truck, phone, current status (On Load / Available), active Load ID if any, completed loads count, and last GPS ping time.

The dashboard auto-refreshes every 60 seconds and also updates in real-time via Socket.IO events (status-updated, pod-uploaded, load-assigned).

Creating New Jobs /jobs/new

This page is Super Admin exclusive. Dispatchers cannot create new loads.

Load Information Section

  • Load ID — Enter manually or click "Auto" to generate a unique ID
  • Rate/Amount — Payment for the load (e.g., $3,500)
  • Broker/Customer — Company name of whoever booked the load
  • Broker Contact, Phone, Email — Contact details (visible only to Super Admin)

Pickup & Drop-off Details

  • Address — Type to search with Google Places autocomplete. Selecting a result auto-fills coordinates.
  • Date — Scheduled pickup/delivery date and time
  • Map Picker — Click the map icon to open an interactive map and visually place the pin
  • Coordinates can also be entered manually (lat/lng fields)

Click "Create Load" to submit. The load appears as Unassigned in the Dashboard Job Board. A duplicate Load ID warning appears if the ID already exists.

Live Tracking /tracking

Interactive Map

  • Driver markers — Each active driver appears as a pin with their name
  • Route polylines — Lines showing planned pickup-to-delivery routes
  • Click a marker to see: driver name, load ID, current status, speed, ETA
  • Map type toggle — Switch between roadmap and satellite views
  • Positions update in real-time as drivers report GPS via Socket.IO

ETA Information

Calculated using straight-line distance at 55 mph default speed. Compared against scheduled delivery date to flag loads as "on-time" or "delayed".

Expenses /expenses

All Expenses Tab

Table of all driver-submitted expenses. Columns: driver, load ID, type, amount, description, receipt, status (Pending/Approved/Rejected), date. Use Approve or Reject buttons per row.

Fuel Analytics Tab

Dedicated fuel analysis: cost per gallon, total fuel spend, monthly trends, per-driver fuel efficiency breakdowns, and gallons consumed. Based on fuel-type expenses with gallons and odometer data.

Messages /messages

Split-Panel Layout

  • Left panel — Driver list with unread count badges
  • Right panel — Active conversation thread
  • Click a driver name to open their chat history

Messaging Features

  • Send text messages to any driver
  • Attach documents/images (files are uploaded and linked)
  • Messages can include Load ID tags for context
  • Real-time delivery via Socket.IO — no need to refresh
  • Timestamps on every message, read receipts
Notifications /notifications

Notification Feed

Chronological list of system events relevant to dispatch operations:

  • Driver status changes (e.g., "Lesline Johnson updated Load LD-001 to In Transit")
  • POD uploads (e.g., "POD uploaded for Load LD-001")
  • Geofence triggers (e.g., "Lesline Johnson arrived at pickup location")
  • Load accepts/declines from drivers
  • System alerts

Click a notification to mark it as read. "Mark All Read" button clears all unread badges. Notifications arrive in real-time via WebSocket.

Data Manager /data

Direct interface to Google Sheets data. Super Admin exclusive.

Sheet Navigation

Left sidebar lists all Google Sheet tabs (Job Tracking, Carrier Database, Payments Table, Status Logs, Job Summary, etc.). Click a tab to load its data into the main table.

CRUD Operations

  • View — Paginated table (5/10/20 rows per page) with full-text search across all columns
  • Edit — Click a row to edit inline. Driver columns render as dropdowns. Save or cancel.
  • Add — "Add Row" button opens a modal with fields matching the sheet headers
  • Delete — Delete button with confirmation dialog. Rows shift up after deletion.

Duplicate Detection

Toggle duplicate mode to highlight rows sharing the same Load ID. Groups are displayed together for easy comparison. Dangerous duplicates (different statuses) are flagged.

Editing sheets directly bypasses business logic (status guards, validation, notifications). Use the Dashboard for normal operations; Data Manager is for administrative corrections.

User Management /users

Creating Users

The "Add User" form at the top requires:

  • Username (required, unique, case-insensitive)
  • Password (required, min 4 characters)
  • Role dropdown: Super Admin, Dispatcher, Driver, Investor
  • Driver Name (for Driver role — links to Google Sheets)
  • Full Name, Email, Company Name (optional)

User Table

  • View all users with role, driver name, email, and creation date
  • Edit — Modify role, name, email (password reset available)
  • Delete — Remove user with confirmation
  • Rate Drivers — 1-5 star rating for driver performance

Creating a Driver user auto-syncs a row to the "Carrier Database" Google Sheet. The Driver Name field is the KEY that links the SQLite account to Google Sheet data — it must match exactly.

Truck Database /trucks

Adding Trucks

  • Unit Number — Unique identifier (e.g., TRK-101)
  • Make, Model, Year, VIN, License Plate — Vehicle details
  • Status — Active, Inactive, Maintenance, or OOS
  • Assigned Driver — Dropdown of unassigned drivers
  • Owner — Dropdown of Investor users (Super Admin only field)

Business Configuration

Per-truck financial settings (expandable section):

  • Purchase Price — For depreciation and investor valuation
  • Title Status — Clean, Lien, or Mixed
  • Insurance, ELD, HVUT, IRP — Monthly/annual fixed costs
  • Driver Pay Daily — Driver compensation rate
  • Maintenance Fund Monthly — Sinking fund target (default $800)

A driver with an active load cannot be reassigned to a different truck. Complete or cancel their load first. Auto-unassigns the driver from the previous truck when reassigned.

Admin Tools /admin/tools

Data quality scanning and cleanup utilities. Super Admin exclusive.

1. Duplicate Load ID Scanner

Scans Job Tracking for rows sharing the same Load ID. Groups duplicates and flags "dangerous" ones (same ID, different statuses). Click to remove the problematic row, keeping the one with the most advanced status.

2. Driver Name Mismatch Scanner

Cross-references driver names across three sources: User Accounts table, Carrier Database sheet, and Job Tracking sheet. Detects: spelling differences, missing accounts, orphan references. One-click Rename action to standardize names.

3. Orphan Record Scanner

Finds SQLite records (expenses, messages, locations) referencing driver names that don't exist in the Carrier Database. Shows count of orphan records per driver name.

4. Stale Location Data Scanner

Identifies GPS records older than 7 days or referencing completed/non-existent loads. Option to retag or clean up stale entries.

Dashboard Operations /dashboard

The Dashboard looks similar to the Super Admin view with these key differences:

What's the Same

  • Full access to KPI cards (Active Loads, Unassigned, Completed, Fleet)
  • All 4 tabs: Job Board, Active Loads, Completed, Fleet & Drivers
  • Can assign, reassign, and cancel load assignments
  • Can force status updates from the Active Loads tab
  • Full search, pagination, and load detail dialogs

What's Different

  • No Revenue Cards — Financial data (Total Revenue, Paid, Pending) is hidden
  • No Broker Details — Broker phone/contact columns are sanitized
  • No Rate Column — Load rates/amounts hidden from raw data views
Daily Dispatcher Workflow

Morning Routine

  • Check Notifications for overnight status changes, POD uploads, geofence triggers
  • Review Dashboard → Job Board for unassigned loads
  • Assign drivers to available loads based on location and fleet availability
  • Check Dashboard → Fleet & Drivers for driver status and availability

Throughout the Day

  • Monitor Tracking map for real-time driver positions and ETAs
  • Respond to driver Messages (questions about loads, addresses, issues)
  • Review and approve driver Expenses (fuel, tolls, food)
  • Handle load reassignments if a driver declines or has issues
  • Check Active Loads tab for status progression

Truck Database Access

Dispatchers can add and edit trucks (including driver assignment). They cannot delete trucks — that requires Super Admin. The Owner field is visible and editable.

Driver

Mobile-optimized interface for drivers on the road. Accept loads, update status, upload POD, track expenses, and communicate with dispatch. No sidebar — uses bottom navigation.

App Layout & Bottom Navigation

The Driver App at /driver uses a mobile-first layout with no sidebar. Navigation is via the bottom tab bar:

📦Loads
📊Status
🔔Alerts
📋Kit
💬Messages

Header Bar

Shows the driver's name, connection status indicator, and logout button. When a new load is dispatched, a prominent assignment banner appears at the top with "View Load" action.

The Driver App uses the Vant Mobile UI framework for touch-friendly controls optimized for phone screens. Desktop users will see the same interface.

Loads Tab

Sub-Tabs

  • Active — Loads currently in progress (Assigned through Delivered)
  • Pending — Dispatched loads awaiting your Accept/Decline response
  • Historical — Completed/delivered loads (read-only history)

Load Cards

Each load appears as a card showing: Load ID, origin → destination, status badge (color-coded), pickup/delivery dates, and notes. Tap to view full details.

Search & Filters

Toggle the filter panel to search by Load ID, origin, destination, or details. Date range filters (From/To) narrow results further.

Load Detail View

Tap a load card to see the full detail page with collapsible sections:

  • Truck Details — Unit #, make/model, year, VIN, plate, photo
  • Pickup Details — Address (with copy and map buttons), date, coordinates
  • Drop-off Details — Same as pickup for the destination
  • Route Map — Interactive Google Map showing the route (expandable to fullscreen)
  • Documents — Uploaded POD and other files with download links
  • Broker Details — Broker name, contact, phone, email

Accept / Decline (Pending Loads Only)

For loads in the Pending tab, two action buttons appear:

  • Accept — Confirms you'll take the load. Status changes to Assigned. Dispatch is notified.
  • Decline — Returns load to Unassigned. Dispatch is notified you declined.
Status Updates (Stepper)

The Status tab shows a visual stepper with the load's progress. Each step requires one tap with a confirmation dialog.

Assigned➜ At Shipper➜ Loading➜ In Transit➜ At Receiver➜ Delivered

How to Update Status

  • The stepper highlights your current position in the flow
  • A single action button shows the next available step (e.g., "Mark as In Transit")
  • Tap the button, confirm in the dialog, and status updates immediately
  • Every status change is logged to the "Status Logs" Google Sheet
  • Dispatch receives a real-time notification for each change

One-Active-Job Rule: You cannot start a new load ("At Shipper") while another load is in active status (At Shipper, Loading, In Transit, At Receiver). You'll see a 409 error: "You already have an active job. Complete it before starting another."

POD Required for Delivery: The "Delivered" button stays disabled until at least one document (proof of delivery) has been uploaded for the load. Upload POD first, then mark Delivered.

Documents & POD Upload

Uploading Documents

  • In the load detail view, find the Documents section
  • Select Document Type: POD, Receipt, Signature, Photo, or Other
  • Choose a file or capture with camera
  • Camera photos are auto-converted to PDF by the server
  • Multiple images can be combined into a single multi-page PDF

Document List

All uploaded documents for the load appear in a list showing: file name, type, upload date, and download/view link. Documents are stored locally in uploads/ and optionally on Google Drive.

Upload your POD as soon as delivery is confirmed. This enables the "Delivered" button and notifies dispatch that proof of delivery is available.

GPS Tracking & Geofence

Automatic GPS Tracking

When you have an active load, the app automatically tracks your position in the background using your phone's GPS. No manual action needed.

  • Reports your position when you move 50+ meters
  • Sends a heartbeat every 60 seconds even when stationary
  • Readings with accuracy worse than 100 meters are discarded
  • Your position appears on dispatch's Tracking map in real-time

Geofence Auto-Triggers

The system automatically detects when you arrive at pickup or delivery locations:

  • Near Pickup (500m): If your status is "Dispatched" or "Assigned", it auto-changes to At Shipper
  • Near Delivery (500m): If your status is "In Transit", it auto-changes to At Receiver
  • You'll see a notification: "You have arrived at the pickup/delivery location"

Distance Warnings

If you're more than 500 km from your target location (pickup when heading there, delivery when in transit), a warning banner appears at the top of the screen showing the distance. This helps catch routing errors early.

GPS tracking requires browser location permissions. If you deny the permission, tracking won't work and dispatch won't see your position. Grant "Always Allow" for best results.

Expenses

Submitting an Expense

  • Tap Expenses in the bottom nav (or use the expense form in load detail)
  • Select the Load from the dropdown (shows your active loads)
  • Choose Type: Fuel, Repair, Maintenance, Wear & Tear, Toll, Food, Other
  • Enter Amount and Description
  • For Fuel: also enter Gallons and Odometer reading
  • Optionally attach a receipt photo (camera or file picker)
  • Tap Submit

Expense History

Below the form, your submitted expenses appear as cards showing: type, amount, load, date, and status (Pending / Approved / Rejected). You can view receipt images on submitted expenses.

Submit fuel expenses at each fuel stop with accurate gallons and odometer readings. This data feeds the company's fuel analytics and IFTA compliance reports.

Messages & Alerts

Messages Tab

  • Chat interface for communicating with dispatch
  • Messages appear as bubbles with timestamps
  • Load ID tags link messages to specific loads for context
  • File attachments supported (image preview in messages)
  • Unread badge on the Messages icon in bottom nav
  • Real-time delivery — messages appear instantly

Alerts Tab (Notifications)

  • Timeline of notifications sent to you specifically
  • Types: load assignments, status change confirmations, geofence alerts, system messages
  • Tap a notification to mark it as read
  • "Mark All Read" button available
  • Unread badge count on the Alerts icon

Kit Tab

Read-only view of your driver profile information pulled from the Carrier Database sheet. Shows personal details, contact info, assigned truck details, and carrier information.

Investor

The truck company owner. Views financial performance, fleet health, tax benefits, and legal documents for owned trucks. Limited to financial dashboards and fleet overview.

Dashboard Overview /investor

After login, the Investor lands on /investor. The sidebar shows only 2 items:

📈Financial Overview/investor
📋My Trucks/trucks

Hero Header

Displays "{Company Name} — Asset Dashboard" (e.g., "KC Trucking LLC — Asset Dashboard"). Shows today's date formatted as "Monday, April 03, 2026".

  • Contact Operations — Email link to operations team
  • Contact Tech Support — Email link to tech support
  • Download Report — Select date range and export financial report
  • Refresh — Reload all dashboard data

Quick Stats Strip

  • Total Revenue — Formatted as $X.XXK or $X.XXM
  • Completed Loads — Total delivered loads for your trucks
  • Fleet Size — Number of trucks you own
  • Avg / Day — 30-day rolling average revenue

All data on the Investor Dashboard is filtered to your owned trucks only. You only see revenue, expenses, loads, and metrics for drivers assigned to trucks registered under your ownership.

Financial Sections

Production Metrics

  • Total Revenue — Sum of all payments (from Payments Table or Job Tracking)
  • Paid vs Pending — Breakdown by payment status
  • Completed Jobs — Total loads delivered
  • Avg Daily Revenue — Last 30 days divided by 30
  • Avg Monthly Owner Earnings — Net revenue spread across months of operation

Revenue Trends

Monthly revenue data showing income trajectory over time. Visualizes growth or decline patterns.

Cash Flow & Projections

  • Net Revenue to Date — Total revenue minus all expenses (driver expenses, maintenance fund services, compliance fees, truck fixed costs)
  • Owner Earnings Estimate — Projected share of net revenue
  • Break-Even Timeline — Progress bar showing when total revenue will cover total investment
  • Business ROI — Return on investment to date
  • Truck Payoff Timeline — Progress bar for each truck

Tax Shield (Section 179)

  • Section 179 Deduction — 100% deductibility of truck purchase prices
  • At-Risk Capital — Max of (purchase price + startup expenses) minus net revenue
  • Write-Off Percentage — 100% (accelerated depreciation)
  • Useful for annual tax planning with your accountant
Fleet & Assets

Asset Overview

  • Purchase Price — Total cost of all owned trucks
  • Current Value — 80% of purchase price (20% flat depreciation)
  • Title Status — Clean, Lien, or Mixed across fleet
  • Total Miles — Aggregate fleet mileage from odometer data
  • Revenue Per Mile — Total revenue divided by total miles
  • Cost Per Mile — Total expenses divided by total miles

Per-Truck Breakdown

Detailed table for each truck showing:

  • Unit number, make/model, year
  • Assigned driver and their status
  • Monthly gross revenue and monthly expenses
  • Estimated annual revenue (monthly net × 12)
  • Total miles driven
  • ROI percentage

My Trucks Page (/trucks)

View-only list of your owned trucks showing: unit number, make/model, year, VIN, plate, status, assigned driver, and photo. You cannot add, edit, or delete trucks — contact the Super Admin for fleet changes.

Tools & Communication

Legal Document Portal

  • View per-truck legal documents uploaded by Super Admin
  • Document types: insurance certificates, vehicle titles, registrations, permits, operating agreements
  • Download documents for your records
  • Organized by truck unit number

Investor Chat

Direct messaging channel with the operations team (Super Admin). Use it for questions about loads, payments, fleet status, or any operational concerns.

Download Reports

Click "Download Report" in the hero header. Select a date range (From / To) and generate a financial summary for the period. Useful for monthly reviews or tax preparation.
