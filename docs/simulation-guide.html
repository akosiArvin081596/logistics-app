LogisX — Complete System Simulation Guide

End-to-end walkthrough from fresh install to daily operations
1 System Initialization

Set up the environment, launch the server, and create the first Super Admin account.

Prerequisites

Before launching LogisX, ensure you have:

RequirementDetails
Node.jsv18+ recommended. Run node -v to verify.
Google Cloud ProjectSheets API v4 and Drive API enabled. Service account created.
service-account-key.jsonDownloaded from Google Cloud Console. Place at project root.
Google SheetCreate a spreadsheet with tabs: Job Tracking, Carrier Database, Payments Table, Status Logs. Share with service account email as Editor.
.env fileAt project root with GOOGLE_MAPS_API_KEY (required) and GOOGLE_DRIVE_FOLDER_ID (optional).

The Google Sheet spreadsheet ID is configured in server.js. Update it if using a different sheet.

First Launch
1

Install dependencies

Run npm install at the project root. This automatically runs cd client && npm install via the postinstall hook, installing both backend and frontend packages.

2

Start the development servers

Open two terminals:

Terminal 1 (Backend): npm run dev — starts Express on port 3000 with nodemon auto-restart.

Terminal 2 (Frontend): npm run dev:client — starts Vite dev server on port 5173 with proxy to backend.

3

Open the application

Navigate to http://localhost:5173 in your browser. The system detects that no users exist and displays the First-Time Setup screen.

For production: run npm run build:client then npm start. Express serves the built SPA from client/dist/ on port 3000.

Create the Super Admin Account
1

Fill the setup form

Enter a Username (e.g., admin), Password (minimum 4 characters), and optionally an Email.

2

Click "Create Super Admin Account"

The system calls POST /api/auth/setup, creates the user in SQLite with role Super Admin, and auto-logs you in.

3

Redirected to Dashboard

You're taken to /dashboard — the operations command center. It will be empty until we create loads and users.

The setup endpoint is only available when no users exist. Once the Super Admin is created, the setup form is permanently hidden and login is shown instead.

Status Progression Driver

The driver progresses through each status using the Status Stepper component. Each step requires a single tap with confirmation.

Step 1: At Shipper

Driver arrives at the pickup location. Tap "Arrived at Shipper" on the status stepper. Status changes to At Shipper.

Step 2: Loading

Truck is being loaded with cargo. Tap "Loading". Status changes to Loading.

Step 3: In Transit

Departing with cargo en route to delivery. Tap "In Transit". Status changes to In Transit. GPS tracking actively reports position to dispatch.

Step 4: At Receiver

Driver arrives at the delivery location. Tap "Arrived at Receiver". Status changes to At Receiver. A POD reminder modal may appear prompting document upload.

Step 5: Delivered

Cargo has been unloaded and delivery is complete. Tap "Delivered". Status changes to Delivered.

One-Active-Job Constraint: A driver cannot start a new load (transition to "At Shipper") if they already have another load in an active state (At Shipper, Loading, In Transit, At Receiver). The system returns a 409 Conflict error. Complete or cancel the current load first.

POD Gate: The "Delivered" button is disabled until at least one document (POD) has been uploaded for the load.

Geofence Auto-Triggers System

LogisX uses GPS geofencing to automatically advance load status when the driver physically arrives at pickup or delivery locations.

How It Works

  • The driver's phone continuously reports GPS coordinates via POST /api/location
  • The server checks if the driver is within a 500-meter radius of either the pickup or delivery coordinates
  • Uses geolib.isPointWithinRadius() for distance calculation

Auto-Trigger Rules

  • Near Pickup: If current status is "Dispatched" or "Assigned" → auto-changes to At Shipper
  • Near Delivery: If current status is "In Transit" → auto-changes to At Receiver
  • Both the driver and dispatch receive notifications when a geofence triggers

GPS Reporting Details

  • Movement threshold: Reports when driver moves 50+ meters
  • Minimum interval: No more than one report per 10 seconds
  • Heartbeat: Reports every 60 seconds even if stationary
  • Accuracy filter: Readings with accuracy >100m are discarded
  • Distance warning: If driver is 500+ km from target, a warning banner appears

Geofence errors are silently caught — they never block the location report. Even if geofence logic fails, the driver's location is still recorded and visible on the tracking map.

POD Upload & Completion Driver
1

Navigate to Documents section

In the Load Detail view, scroll to the Documents section, or use the Status tab's document upload area.

2

Upload proof of delivery

Choose Document Type = "POD" from the dropdown. Either capture a photo with the camera or select a file. Multiple images can be combined into a single PDF.

3

Server processing

The server converts camera photos to PDF via pdfkit, saves locally to uploads/, and optionally uploads to Google Drive. OCR is attempted on receipts via tesseract.js (silent failure if OCR fails).

4

Complete the delivery

With POD uploaded, the "Delivered" button on the status stepper becomes enabled. Tap to finalize. The load moves to Delivered and eventually POD Received.

Dispatch receives a pod-uploaded Socket.IO event and sees the document in the Dashboard's completed loads section.

Investor Dashboard Walkthrough Investor

Log in with an investor account (e.g., kevin). You're directed to /investor.

Hero Header

Shows "KC Trucking LLC — Asset Dashboard" (company name from account). Today's date. Action buttons: Contact Operations, Contact Tech Support, Download Report (with date range), Refresh.

Quick Stats Strip

  • Total Revenue — sum of all payments for investor's drivers
  • Completed Loads — count of delivered/completed jobs
  • Fleet Size — number of trucks owned by this investor
  • Avg / Day — 30-day rolling average revenue

Production Metrics

Total jobs, completed jobs, completion rate, revenue per job, average monthly owner earnings. All filtered to the investor's owned trucks and their assigned drivers.

Revenue Trends

Monthly revenue data visualization showing income trajectory over time.

Asset Overview

Fleet valuation (aggregate purchase price), current value (80% of purchase — 20% flat depreciation), total fleet miles, revenue per mile, cost per mile.

Per-Truck Breakdown

Table showing each truck: unit number, make/model, assigned driver, status, monthly gross, monthly expenses, estimated annual revenue, total miles, ROI percentage.

Cash Flow & Projections

Net revenue to date (revenue − all expenses), owner earnings estimate, break-even timeline with progress bar, ROI calculation.

Tax Shield (Section 179)

100% deductibility of truck purchase price under Section 179. Shows at-risk capital: max of (purchase price + startup expenses) minus net revenue. Write-off percentage: 100%.

Legal Document Portal

View and download per-truck legal documents: insurance, registration, titles, permits. Super Admin uploads these; investor can view/download.

Investor Chat

Direct messaging with operations team for questions about loads, payments, or fleet status.

All financial data is filtered by the investor's owned trucks. The investor only sees revenue, expenses, and loads associated with drivers assigned to their trucks.

7 Data Management & Admin Tools

Super Admin tools for data quality, sheet management, and system health.

Data Manager Super Admin

Navigate to Data Manager (/data) in the sidebar. This is the direct interface to Google Sheets data.

Sheet Navigation

Left sidebar lists all Google Sheet tabs (Job Tracking, Carrier Database, Payments Table, Status Logs, etc.). Click a tab to load its data.

Data Table Operations

  • View: Paginated table with search across all columns
  • Edit: Click a row to edit inline. Save or cancel changes.
  • Add: "Add Row" button opens a modal with fields based on sheet headers
  • Delete: Delete button with confirmation modal (Super Admin only)
  • Duplicates: Toggle duplicate detection to find duplicate Load IDs

Editing Google Sheets directly bypasses business logic (status guards, validation). Use the Dashboard for normal operations; Data Manager is for corrections and administration.

Admin Tools Super Admin

Navigate to Data Tools (/admin/tools) in the sidebar.

1. Duplicate Load ID Scanner

Scans Job Tracking for duplicate Load IDs. Groups duplicates and flags "dangerous" ones (same ID with different statuses). One-click remove for problematic rows.

2. Driver Name Mismatch Scanner

Cross-references driver names across: User Accounts, Carrier Database sheet, and Job Tracking sheet. Detects spelling inconsistencies, missing accounts, and name conflicts. One-click rename to fix mismatches.

3. Orphan Record Scanner

Finds SQLite records (expenses, messages, locations) referencing driver names that don't exist in the Carrier Database. Shows count per orphan driver.

4. Stale Location Data Scanner

Identifies GPS location records older than 7 days or referencing completed/non-existent loads. Option to retag or clean up stale entries.

8 IFTA Compliance

State-by-state mileage reporting for fuel tax compliance.

IFTA Mileage Report Super Admin Dispatcher

What is IFTA?

The International Fuel Tax Agreement requires carriers to report miles driven per state for quarterly fuel tax filings. LogisX automatically calculates this from GPS data.

How It Works

  • GPS location history is analyzed by the server (GET /api/compliance/ifta)
  • Distance between consecutive GPS points is calculated using geolib.getDistance()
  • Gaps greater than 30 minutes are treated as separate trips
  • Distances greater than 50 miles between points are filtered out (GPS drift)
  • Each GPS midpoint is classified by state using built-in bounding boxes (~24 US states)
  • Miles are summed per state per driver

Report Output

  • Total miles across the fleet
  • Per-state breakdown with miles, percentage, and contributing drivers
  • Date range defaults to last 2 months (configurable)
  • Data can be used for quarterly IFTA filings

Covered states include: TX, OK, LA, AR, NM, MS, AL, TN, GA, FL, MO, KS, CO, AZ, CA, NV, IL, IN, OH, PA, NY, NC, SC, VA, and more.

9 Live Tracking

Real-time fleet visualization on an interactive map.

Real-Time Tracking Map Super Admin Dispatcher

Navigate to Tracking (/tracking) in the sidebar.

Map Features

  • Driver markers — Each active driver appears as a pin on the Google Map
  • Route polylines — Lines showing the planned route from pickup to delivery
  • Info windows — Click a driver marker to see: name, current status, speed, active load ID, ETA
  • Map type toggle — Switch between roadmap and satellite views
  • Real-time updates — Positions refresh as drivers report GPS via Socket.IO

ETA Calculation

  • Uses geolib.getDistance() from current position to destination
  • Default speed: 24.587 m/s (~55 mph) when GPS speed is unreliable
  • Compares ETA vs scheduled delivery date to flag "on-time" or "delayed"
